# 会话交接摘要目录

本目录存放跨会话接力用的短交接摘要，记录"当前做到哪里、未提交什么、下一步怎么接"。

使用规则：

- 新会话如果是继续上一轮开发、修复、提交或评审，先读取本目录最新的交接摘要。
- 交接摘要只写当前接力需要的信息，不复制 `rules/`、`mechanisms/`、`dev-plans/` 中已有的完整内容。
- 文件命名建议：`YYYY-MM-DD-主题-handoff.md`；多 Agent 协作时追加 Agent 后缀，例如 `YYYY-MM-DD-主题-handoff-claude.md`。
- 交接摘要不是长期机制文档；当内容变成稳定机制，应迁移或沉淀到 `mechanisms/`。
- 当内容变成必须长期遵守的规则，应迁移或沉淀到 `rules/`。
