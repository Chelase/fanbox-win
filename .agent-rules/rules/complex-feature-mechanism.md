# 复杂跨模块功能机制文档强制生成

## 用途

当实现或修改涉及多个模块穿插交互、调用链路复杂、或后期维护风险较高的功能时，
必须在实施过程中同步生成或更新对应的机制文档，确保模块协作方式被显式记录，
防止后续 Agent 和开发者因缺乏链路理解而引入回归或破坏现有行为。

## 触发条件

满足以下任一条件即触发本规则：

1. **跨模块调用**：功能实现涉及 2 个及以上模块之间的数据流、API 调用或事件传递。
2. **非直链交互**：功能流程中存在间接调用、异步消息、回调、事件订阅/发布、中间件代理等非直观链路。
3. **状态变更连锁**：一次操作会触发多个模块中的状态变更，且这些变更之间存在时序或因果依赖。
4. **公共数据契约变更**：修改了多个模块共享的数据结构、接口协议、枚举定义或事件格式。
5. **既往复杂历史**：同一功能区域在过去 3 个月内被不同 Agent 修改过 2 次以上，且没有对应的机制文档。
6. **Agent 自主判断**：Agent 在分析阶段发现理解该功能需要同时读取 3 个以上模块的文件才能厘清全貌。

## 执行方式

1. **实施前**：Agent 在完成需求分析和受影响范围评估后，若判定触发本规则，应先撰写机制文档的骨架（结论 + 涉及对象 + 运行链路草案），再开始编码。
2. **实施中**：随着实现深入，持续更新运行链路、修改点和使用点，确保文档与代码同步。
3. **实施后**：最终检查机制文档的完整性和准确性，确认七段结构完整、链路图与实际行为一致、索引已更新。

## 文档存放位置

- 机制文档写入 `.agent-rules/mechanisms/`，按 `mechanisms/README.md` 中的七段强制结构编写。
- 索引同步更新：`mechanisms/README.md` 清单 + `.agent-rules/README.md` 的机制索引段。

## 违反后果

未按本规则生成机制文档的复杂跨模块功能，后续维护或迭代时若因链路理解缺失导致线上故障、回归或重复排查浪费时间，视为规则违反。
